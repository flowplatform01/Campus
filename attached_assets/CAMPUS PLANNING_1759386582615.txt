Arch org :
1. Frontend (apps/client & apps/admin)
Client App (React + TypeScript)
Pages, Components, Data-access layer
Routes: Social tab (currently placeholder)
State management: Hooks/Contexts
Makes API calls to Server
Admin Dashboard (React + TypeScript, similar to Client)
Pages for management
Features: Moderation, Analytics
Makes API calls to Server

2. Backend (apps/server/src)
Core Server (Node.js + TypeScript)
Entry point: index.ts
Modules: users, academics, payments, communication, media, social (placeholder)
Controllers: API endpoints (e.g., /posts, /grades)
Services: Business logic (validation, processing, etc.)
Entities: Prisma models
Middlewares: Auth, Rate-limiting
Utils: Helpers (e.g., WebSockets)
Validations: Input checks
Workers
Background jobs (queues, crons, media processing)
Webhook handlers (e.g., Stripe events)
Async tasks via Redis queues
Configs
db.config.ts: Neon Postgres connection
server.config.ts: Ports, env vars
storage.config.ts: R2 setup
notifications.config.ts: SendGrid/FCM
integrations.config.ts: API keys

3. Database & Storage
Neon Postgres (via Prisma)
schema.prisma: Models (User, Post, School, etc.)
Migrations & seeds
Structured data: metadata, records
Limits: 0.5 GB free storage, 10 GB egress/month
Redis
Caching (feeds, results, timelines as sorted sets)
Session storage for auth
Pub/Sub: real-time events (e.g., comments)
Queues for workers
Free tier: ~30 MB memory
Cloudflare R2 (Object Storage)
Media files: images, videos, documents
SDK: AWS S3 compatible
Signed URLs for uploads/downloads
Free tier: 10 GB, unlimited egress

4. External Integrations (server/src/integrations)
Stripe (Payments)
SDK calls from Server
Webhooks handled by Workers
Pricing: 2.9% + $0.30 per transaction
SendGrid / Resend (Emails & Notifications)
SDK sends from Communication module
Free tier: 100–3k emails/month
PostHog (Analytics)
Tracks client/server events
Free tier: 1M events/month
Other Integrations (Google Calendar, FCM, etc.)
API pulls & webhooks
Used for scheduling & push notifications
